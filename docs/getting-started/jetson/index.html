Jetson Nano

Running DEC Potala on Jetson Nano

Nvidia Jetson Nano is based on ARM Cortex A57 CPU that can run both 64bit and 32bit ARM instructions. The Jetson Nano™ Developer Kit refers to both HW and SW, as discussed on Jetson Nano Developer Kit

Prerequisites

  • Nvidia Jetson Nano

  • Picamera Module v2.1, Sony IMX219 8-megapixel sensor

  • [Optional] USB storage: flash drive, SSD drive, or hard drive

Installation

If you are not familiar with Nvidia Jetson Nano, please visit Getting Started With Jetson Nano Developer Kit to get acquaintance with your GPU computer. Follow the beginner instructions that includes: introduction, prepare for setup, write image to the microSD card, setup and first boot, next steps, and troubleshooting.

Jetson Nano Operating System (i.e., Ubuntu) is bundled in the Jetpack SDK that also contains developer tools, libraries and APIs, samples, and documentation needed to develop AI application using Nvidia CUDA.

[Recommended] the instruction on the “write image to the microSD card” step, asks to download the SD Card image in zip file (1. Download the Jetson Nano Developer Kit SD Card Image) by clicking on the link. NOTE: the link may point to an old version file. As of this writing, the latest version file is jetson-nano-sd-r32.2.1.zip which contains to Jetpack version 4.2.2, and can be & should be downloaded from: - https://developer.nvidia.com/embedded/jetpack

Also note that there are two ways of flashing (a.k.a, installing OS on Jetson Nano):

  • Recommended - download the SD Card image to a computer or laptop, and flash the image to the SD card using, for example, Etcher. The SD Card image is refered by NVIDIA as JetPack SDK and the latest version as of this writing is version 4.2.2; NOTE: using version 4.2.1 or later is important because since 4.2.1 it included a beta version of NVIDIA Container Runtime with Docker integration for the Jetson platform.

  • The second way to setup OS on Jetson Nano is to use the NVIDIA SDK Manager; this method also to be perform from a computer/laptop, seems to be more complicated and involves SDK Manager GUI. Has not been attempted.

Step-by-step instructions

  • From a computer or laptop, if not already downloaded, go to https://developer.nvidia.com/embedded/jetpack and download the latest version of Jetpack version 4.2.2. As of this writing, the latest version file is jetson-nano-sd-r32.2.1.zip which contains to Jetpack version 4.2.2. Therefore, the following instructions are based on version 4.2.2.

  • From the computer and use the downloaded .zip file, install the operating system image on an SD card. The steps are exactly the same as the Raspberry Pi as described in Installing operating system images:

    • check the SD card requirements but if your SD card is not greater than 32GB, you should be OK to proceed
    • Download balenaEtcher and install on your computer; balenaEtcher is a graphical tool to install OS to SD card that works on Mac OS, Linux and Windows.
    • Insert the SD card into the corresponding slot on your computer
    • Follow the Installing operating system images

  • While the SD card is being flashed, prepare your Jetson Nano BUT don’t connect the power supply yet. Connect the HDMI cable to a monitor, mouse & keyboard to USB port, get the wifi SSID and password ready, and setup the camera. For instructions on how to connect the Pi-camera, please visit Jetson Nano + Raspberry Pi Camera. USB camera will be discussed in the future.

  • When the SD card is flashed, eject from computer & insert it into the Jetson Nano to start first booting. In case you can’t find where the micro SD card port is, go to Getting Started With Jetson Nano Developer Kit - Setup and First Boot

  • Continue to the next step, “Setup Jetson Nano”

Setup Jetson Nano

  • After first boot, follow the screen instructions to setup the OS

  • Login using the login id/password you provided during OS setup, a step above. The login id should be: pi for the preconfigured scripts to work later.

  • Check kernel version for your own info

    uname -a 
Linux pi-desktop 4.9.140-tegra #1 SMP PREEMPT Wed Mar 13 00:32:22 PDT 2019 aarch64 aarch64 aarch64 GNU/Linux

  • Jetpack 4.2.2 and newer comes preinstalled with NVIDIA Container Runtime; NVIDIA Container Runtime is a GPU aware container runtime, compatible with the Open Containers Initiative (OCI) specification used by Docker, CRI-O, and other popular container technologies. It simplifies the process of building and deploying containerized GPU-accelerated applications. NVIDIA Container Runtime supports container technologies like Docker so developers can wrap their GPU-accelerated applications along with its dependencies into a single package that is guaranteed to deliver the best performance on NVIDIA GPUs, regardless of the deployment environment. It includes a container runtime library and utilities to automatically configure containers to leverage NVIDIA GPUs.

    Ref: https://github.com/nvidia/nvidia-docker/wiki/nvidia-container-runtime-on-jetson

    Nvidia Container Runtime
    Source: Nvidia

  • Check the NVIDIA Container Runtime is installed & running:

    sudo docker info | grep nvidia

    Output:

    Runtimes: nvidia runc

  • Upgrade Docker runtime - JetPack 4.2.2 also comes with docker runtime v18.09.7; However, NVIDIA Container Toolkit needs docker runtime v19.03 or higher, that supports NVIDIA GPUs natively as devices in the Docker runtime. So usage of nvidia-docker2 packages is no longer need and can be deprecated, as discussed in https://github.com/NVIDIA/nvidia-docker#quick-start.

    docker --version
Docker version 18.09.7, build 2d0083d

    Follow official docker instructions at https://docs.docker.com/install/linux/docker-ce/ubuntu/

    • Uninstall docker

      sudo apt-get remove docker docker-engine docker.io containerd runc

    • Install using convenient script

      sudo apt-get update
sudo apt-get install -y curl
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

    • to use Docker as a non-root user, adding the user to the “docker” group:

      sudo usermod -aG docker <userID>

      Need to reboot for the command to be active

  • After reboot, check version is correct

    docker --version
Docker version 19.03.2, build 6a30dfc

  • Assuming you have connect the Pi-camera to MIPI-CSI while waiting for the SD to be flashed above, check if the camera is mounted correctly and working, by opening a terminal and execute the following to make sure the file exists:

    ls /dev/video0

    Note: the Jetson board supported both MIPI-CSI & USB cameras. The MIPI-CSI, (Mobile Industry Processor Interface - Camera Serial Interface), is for high speed transmission between cameras and host devices, straight to the processor, unlike USB stack with a lot of overhead. For more detailed information, see Jetson Nano + Raspberry Pi Camera. The Pi-camera is connected to CSI interface. The USB camera will be discussed in the future.

  • Check camera streaming - Note that decode, vic and display functionality can be accessed from software using the associated gstreamer plugins available as part of the GStreamer version 1.0 based accelerated solution in L4T. See Github for how to configure gst-launch-1.0

    gst-launch-1.0 nvarguscamerasrc \
! 'video/x-raw(memory:NVMM),width=3820, height=2464, framerate=21/1, format=NV12' \
! nvvidconv flip-method=2 \
! 'video/x-raw,width=960, height=616' \
! nvvidconv ! nvegltransform ! nveglglessink -e

  • Finally, update the Jetson Nano to get the latest OS patches

    sudo apt-get update
sudo apt-get dist-upgrade

  • Now your Jetson Nano host is ready for AI applications
